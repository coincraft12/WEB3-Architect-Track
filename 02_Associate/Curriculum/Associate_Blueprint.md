# Associate Blueprint (Regulation-Aligned)

기준 문서: `00_Governance/Qualification_Management_and_Operation_Regulations.md`

---

## 1. 과정 정의

| 항목 | 내용 |
|---|---|
| 등급 | Associate |
| 응시자격 | WEB3 구조 설계자 Basic 취득자 (제7조) |
| 필기 검정과목 | WEB3 아키텍처 분석 (제6조) |
| 실기 검정과목 | 권한 통제 설계 / 자금 흐름 기반 리스크 평가 (제6조) |

---

## 2. 목표 역량 (제4조·제5조 직결)

**제4조 직무내용:**
- 서비스/프로젝트의 구조를 WEB3 관점에서 해석한다
- 사용자 온보딩·자산흐름·권한/책임 구조를 점검한다
- 위험요소를 식별한다

**제5조 검정기준:**
- 시나리오에서 역할·권한·자산흐름을 파악한다
- 구조적 취약점 및 운영 리스크를 식별한다
- 개선 방향을 제시한다

---

## 3. Basic과의 구분 원칙

Basic은 WEB3의 기초 어휘를 가르친다.
지갑·주소·트랜잭션·가스·스마트컨트랙트의 개념, 그리고 트랜잭션 단위의 구조 읽기(S-A-A-R).

**Associate가 Basic과 다른 것은 분석 단위의 크기가 아니다.**
분석 대상 자체가 다르다.

| 구분 | Basic | Associate |
|---|---|---|
| 배우는 것 | WEB3 기초 개념 (지갑/트랜잭션/컨트랙트) | WEB3 서비스 아키텍처 패턴 (Multisig/Timelock/Proxy/Bridge 등) |
| 분석 단위 | 트랜잭션 1건 | 서비스/프로젝트 전체 구조 |
| 분석 행위 | 읽는다 | 평가하고 개선 방향을 제시한다 |
| 검정 방식 | 필기만 | 필기 + 실기 |

Associate 실기(권한 통제 설계 / 자금 흐름 기반 리스크 평가)를 수행하려면,
Basic에 없는 아키텍처 패턴 지식이 전제되어야 한다.

- Multisig / Timelock / RBAC / Proxy-Upgrade를 모르면 권한 통제 설계를 할 수 없다.
- AMM / Lending / Bridge / Custody 구조를 모르면 자금 흐름 리스크를 평가할 수 없다.

이 전제 지식이 필기 과목(WEB3 아키텍처 분석)의 핵심 학습 내용을 구성한다.

---

## 4. 커리큘럼 구조

검정과목 3개 = 모듈 3개. 모듈 간 학습 의존 관계는 아래와 같다.

```
Module 1 (필기)
  ├─ 권한 통제 메커니즘 학습
  │     └─→  Module 2 (실기 1): 권한 통제 설계
  └─ 자금/자산 흐름 구조 학습
        └─→  Module 3 (실기 2): 자금 흐름 기반 리스크 평가
```

---

### Module 1: WEB3 아키텍처 분석 (필기 대응)

필기 과목이나, 단순 개념 암기가 아니다.
실기 두 과목의 전제 지식을 포함하며, 서비스 전체 구조를 분석하는 방법론을 확립한다.

**학습 내용:**

1. **권한 통제 메커니즘**
   Multisig / Timelock / RBAC / Proxy-Upgrade 패턴
   → 각 메커니즘이 무엇이고, 어떤 목적으로 사용되는지

2. **자산·자금 흐름 구조**
   AMM / Lending / Bridge / Custody 서비스 유형별 구조
   → 각 서비스 유형에서 자산이 어떻게 입금·보관·출금·변환되는지

3. **서비스 단위 구조 분석**
   온보딩 구조 / 자산흐름 경로 / 권한·책임 구조를 서비스 전체에서 통합 분석하는 방법론

---

### Module 2: 권한 통제 설계 (실기 1 대응)

Module 1에서 배운 메커니즘을 가지고, 실제 시나리오에서 평가하고 설계한다.

**학습 내용:**

1. **권한 통제 취약점 유형과 통제안 도출**
   단일 관리자 집중 / 무한 approve / 업그레이드 권한 집중 / 역할 미분리
   → 각 유형의 취약점에서 통제안을 도출하는 방법론

2. **상황별 메커니즘 선택 기준**
   언제 Multisig인가 / 언제 Timelock인가 / 역할 분리 설계 기준

3. **산출물 작성 실습**
   Authority Control Matrix 작성 방법 및 실기 형식 적용

---

### Module 3: 자금 흐름 기반 리스크 평가 (실기 2 대응)

Module 1에서 배운 서비스 구조를 가지고, 실제 시나리오에서 리스크를 평가한다.

**학습 내용:**

1. **자금 흐름 추적 및 리스크 식별**
   입금 → 보관 → 출금 / 교환 / 브릿지 경로별 리스크 식별 방법

2. **서비스 유형별 리스크 특성**
   AMM / Lending / Bridge / Custody 각 유형의 구조적 리스크 특성

3. **리스크 평가 및 개선 방향 도출**
   리스크 심각도 분류 기준, 개선 방향 도출 방법론

4. **산출물 작성 실습**
   Fund Flow Risk Sheet 작성 방법 및 실기 형식 적용

## 5. 실기 산출물

| 산출물 | 대응 검정과목 | 내용 |
|---|---|---|
| Authority Control Matrix | 권한 통제 설계 | 서비스의 역할·권한 현황 및 취약점·통제안 정리 |
| Fund Flow Risk Sheet | 자금 흐름 기반 리스크 평가 | 서비스의 자금 흐름별 리스크 식별 및 개선 방향 |

---

## 6. Professional 경계

- Associate는 **기존 구조의 평가와 개선 방향 제시**까지를 범위로 한다.
- 설계 목표 설정, 거버넌스 구조 설계, 규제 대응 설계는 Professional 범위이다.
- Associate 검정에서 "설계"라는 표현(권한 통제 설계)은 설계안 전체를 처음부터 구성하는 것이 아니라, 주어진 구조에서 취약점을 식별하고 통제 방향을 제시하는 것을 의미한다.
