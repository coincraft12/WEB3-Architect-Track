# 18강. 자금 흐름 리스크 유형 분류: 어떤 구간에서 어떤 리스크가 발생하는가

## 1) 이 강의의 한 문장
**자금 흐름 리스크는 서비스 유형별 구조적 특성에서 오는 리스크와, 컨트랙트 상호작용에서 발생하는 기술적 리스크로 분류되며, 어떤 구간에 어떤 유형의 리스크가 있는지를 매핑해야 Fund Flow Risk Sheet를 완성할 수 있다.**

## 2) 강의 목표 (수강생이 끝나고 할 수 있어야 하는 것)
이 강의를 마치면 수강생은:

1. 자금 흐름 리스크의 두 가지 분류 체계(서비스 유형별 / 컨트랙트 상호작용)를 설명할 수 있다.
2. AMM/Lending/Bridge/Custody 각 서비스 유형의 핵심 리스크를 나열할 수 있다.
3. 재진입 공격, 플래시론, 가격 조작의 개념과 발생 조건을 설명할 수 있다.
4. 자금 흐름 구간별로 리스크 유형을 매핑하는 방법을 말할 수 있다.

---

## 3) 자금 흐름 리스크 분류 체계

자금 흐름 리스크는 두 가지 차원으로 분류한다.

```
자금 흐름 리스크
├─ A. 서비스 유형별 구조적 리스크
│   ├─ AMM 리스크
│   ├─ Lending 리스크
│   ├─ Bridge 리스크
│   └─ Custody 리스크
│
└─ B. 컨트랙트 상호작용 리스크
    ├─ 재진입 공격 (Reentrancy)
    ├─ 플래시론 공격 (Flash Loan Attack)
    └─ 가격 조작 (Price Manipulation)
```

두 차원은 독립적이지 않다. 서비스 유형 리스크가 컨트랙트 상호작용 리스크의 발현 조건을 만들기도 한다.

---

## 4) A. 서비스 유형별 구조적 리스크

### AMM 리스크

**A-1: 비영구적 손실 (Impermanent Loss)**
```
발생 조건: 유동성 공급 후 두 자산의 가격 비율이 크게 변화
발생 구간: 보관 구간 (유동성 공급 후)

예시:
- ETH/USDC 풀에 50:50 비율로 유동성 공급
- ETH 가격 2배 상승 → 풀은 자동으로 ETH를 팔아 USDC 구입
- 결과: 단순 보유 대비 손실 발생 (수수료로 보전되지 않으면 손실)

리스크 수준: 자산 변동성이 클수록 증가
```

**A-2: 유동성 고갈 (Liquidity Drain)**
```
발생 조건: 대규모 LP 동시 출금 또는 유동성 풀 자산 대량 유출
발생 구간: 출금 구간

예시:
- 대형 LP가 전체 유동성의 70%를 동시 인출
- 남은 유동성으로는 정상적 스왑 불가 (슬리피지 폭발)
- 소규모 LP의 출금도 어려워짐

리스크 수준: 유동성 집중도 높을수록 증가
```

**A-3: 가격 슬리피지 리스크**
```
발생 조건: 낮은 유동성 상태에서 대규모 스왑 실행
발생 구간: 스왑 구간

예시:
- 유동성이 낮은 풀에서 큰 스왑 실행
- x*y=k 공식에 의해 출력 가격이 크게 불리해짐
- 프론트러닝(MEV)에 의해 추가 손실 발생 가능

리스크 수준: 유동성 대비 스왑 규모 비율에 비례
```

**A-4: Treasury 중앙화 리스크**
```
발생 조건: 수수료가 단일 통제 주체에게 집중
발생 구간: 수수료 흐름 구간

예시:
- 스왑 수수료 100%가 팀 단일 EOA로 전송
- 해당 EOA 탈취 시 수수료 전액 유출

리스크 수준: Treasury 통제 구조에 따라 다름
```

---

### Lending 리스크

**L-1: Oracle 가격 조작 (Oracle Price Manipulation)**
```
발생 조건: 담보 가격 Oracle이 단일 소스이거나 조작 가능한 AMM 기반
발생 구간: 담보 평가 구간 (가격 조회 구간)

예시:
- 낮은 유동성 AMM을 가격 Oracle로 사용
- 공격자가 대규모 스왑으로 AMM 가격 조작
- 조작된 가격 기준으로 과도한 대출 실행
- 즉시 상환 없이 탈출 → 프로토콜 손실

리스크 수준: Oracle 소스 단일화 및 유동성 낮을수록 증가
```

**L-2: 청산 부족 (Under-Collateralization)**
```
발생 조건: 담보 가격 급락 시 청산자가 없거나 청산이 느릴 때
발생 구간: 청산 구간

예시:
- 담보 자산 가격 50% 급락 (블랙 스완)
- 청산자가 없거나 가스비 폭등으로 청산 지연
- 담보 가치가 부채보다 낮아져 프로토콜 손실

리스크 수준: 담보 자산 변동성, 청산 인센티브, 네트워크 혼잡도에 따라 다름
```

**L-3: 금리 리스크**
```
발생 조건: 자산 이용률이 급격히 변화하거나 금리 모델 오설정
발생 구간: 보관 및 이자 계산 구간

예시:
- 급격한 인출로 이용률 100% 도달 → 금리 폭등
- 차입자 상환 불가 → 공급자 자산 묶임

리스크 수준: 금리 모델 설계와 이용률 관리에 따라 다름
```

**L-4: 연쇄 청산 (Liquidation Cascade)**
```
발생 조건: 다수의 포지션이 비슷한 LTV에 근접해 있을 때 가격 하락
발생 구간: 청산 구간

예시:
- 담보 가격 하락 → A 청산 → 담보 자산 추가 판매 → 가격 추가 하락 → B, C 청산
- 연쇄적으로 청산이 발생하며 가격 폭락 가속화

리스크 수준: 포지션 집중도와 담보 자산 유동성에 비례
```

### Bridge 리스크

**B-1: 검증자 담합 (Validator Collusion)**
```
발생 조건: 브릿지 검증자 임계값 이하의 검증자가 담합 가능한 구조
발생 구간: 검증 구간

예시:
- 5-of-9 검증자 브릿지
- 5명이 담합 → 가짜 Mint 트랜잭션 승인
- 원본 체인의 Lock 자산 없이 대상 체인에서 토큰 발행

리스크 수준: 검증자 탈중앙화 정도, 임계값 높이에 따라 다름
```

**B-2: 자산 불균형 (Asset Imbalance)**
```
발생 조건: 한쪽 체인에서 Lock이 대규모로 일어났지만 Mint가 지연될 때
발생 구간: Lock-Mint 구간

예시:
- Ethereum에서 10,000 ETH Lock
- 검증자 장애로 Polygon에서 wETH Mint 지연
- 사용자 자산이 브릿지 컨트랙트에 묶임

리스크 수준: 검증자 가용성과 장애 복구 절차에 따라 다름
```

**B-3: 래핑 토큰 담보 리스크**
```
발생 조건: 브릿지 래핑 토큰이 다른 프로토콜에서 담보로 사용될 때 브릿지 해킹 발생
발생 구간: 대상 체인 사용 구간

예시:
- Ethereum → Polygon: wETH 발행
- wETH를 Polygon DeFi에서 담보로 사용
- 브릿지 해킹 → wETH 가치 0 → Polygon DeFi 전체 담보 무력화

리스크 수준: 래핑 토큰의 DeFi 사용 규모에 비례
```

**B-4: 스마트 컨트랙트 취약점 (Lock 컨트랙트)**
```
발생 조건: 원본 체인 Lock 컨트랙트에 취약점이 존재할 때
발생 구간: Lock 보관 구간

예시:
- Wormhole 해킹: Lock 컨트랙트 취약점으로 서명 우회
- 공격자가 Lock 없이 Mint 실행 → 대규모 자산 탈취

리스크 수준: Lock 컨트랙트 코드 감사 품질에 따라 다름
```

### Custody 리스크

**C-1: Hot Wallet 해킹**
```
발생 조건: Hot Wallet이 인터넷에 연결되어 있어 해킹 가능
발생 구간: Hot Wallet 보관 구간

예시:
- Hot Wallet 개인키 유출 → 즉시 자산 이동 가능
- Hot Wallet 비율이 높을수록 위험 자산 규모 증가

리스크 수준: Hot/Cold 비율과 키 관리 체계에 따라 다름
```

**C-2: 내부자 위협 (Insider Threat)**
```
발생 조건: 자산 이동 권한을 가진 내부 직원의 악의적 행동
발생 구간: 출금 처리 구간

예시:
- 출금 승인 권한 보유자가 허위 출금 요청 처리
- MPC 서명 참여자가 공모하여 불법 이동

리스크 수준: 내부 통제 절차와 다중 서명 임계값에 따라 다름
```

**C-3: 운영 정지 리스크**
```
발생 조건: Custody 서비스가 규제 조치, 해킹, 파산 등으로 운영 중단
발생 구간: 보관 및 출금 구간

예시:
- Custodial 서비스 파산 → 사용자 자산이 파산 재단에 귀속
- 규제 기관 조치 → 자산 동결

리스크 수준: Custody 서비스의 법적 구조와 재정 건전성에 따라 다름
```

## 5) B. 컨트랙트 상호작용 리스크

### 재진입 공격 (Reentrancy Attack)

**개념:**
```
컨트랙트가 외부 주소로 ETH 또는 토큰을 전송할 때,
수신 컨트랙트가 전송 중에 다시 원래 컨트랙트를 호출하여
잔액이 업데이트되기 전에 추가 인출하는 공격.
```

**발생 조건:**
```
1. 컨트랙트가 외부 호출 전에 상태를 업데이트하지 않을 때
2. 수신자가 컨트랙트 주소일 때 (fallback/receive 함수 실행 가능)
3. 상태 업데이트와 자산 전송 순서가 잘못될 때

취약한 패턴:
function withdraw(uint amount) {
    require(balance[msg.sender] >= amount);
    (bool success, ) = msg.sender.call{value: amount}(""); // 전송 먼저
    balance[msg.sender] -= amount; // 상태 업데이트 나중 ← 취약점
}
```

**발생 가능 구간:**
- 출금 구간 (가장 빈번)
- 청산 구간
- 수수료 수취 구간

**역사적 사례:** The DAO 해킹 (2016, 360만 ETH 손실)

**방어 패턴:**
```
1. Checks-Effects-Interactions 패턴:
   - Check: 조건 확인
   - Effect: 상태 업데이트 먼저
   - Interaction: 외부 호출 마지막

2. ReentrancyGuard 사용 (nonReentrant modifier)
```

### 플래시론 공격 (Flash Loan Attack)

**개념:**
```
플래시론: 담보 없이 한 트랜잭션 내에서 대규모 자산을 빌리고
          같은 트랜잭션 내에서 상환해야 하는 대출.
          상환 못하면 트랜잭션 전체 롤백.

플래시론 공격: 플래시론으로 빌린 대규모 자금을 이용하여
              가격 조작, 거버넌스 공격, 담보 조작 등을 실행하는 것.
```

**발생 조건:**
```
1. 프로토콜이 플래시론 가능한 자산을 보유
2. 프로토콜 내 계산이 블록 내 순간적 상태에 의존
3. 가격 오라클이 조작 가능한 AMM에 의존

플래시론 공격 순서:
① 플래시론으로 대규모 자산 차입
② 차입 자금으로 Oracle 가격 조작 (AMM 대규모 스왑)
③ 조작된 가격 기준으로 프로토콜에서 과도한 대출 실행
④ 프로토콜에서 받은 자산으로 플래시론 상환
⑤ 이익 확보 후 탈출
```

**발생 가능 구간:**
- Oracle 가격 조회 구간 (가장 빈번)
- 담보 평가 구간
- 거버넌스 투표 구간 (투표권 순간 확보)

**방어 방법:**
```
1. 시간 가중 평균 가격(TWAP) Oracle 사용
2. Chainlink 같은 외부 독립 Oracle 사용
3. 플래시론을 이용한 가격 조작에 저항하는 Oracle 설계
```

---

### 가격 조작 (Price Manipulation)

**개념:**
```
프로토콜이 의존하는 가격 정보(Oracle)가 공격자의 행동으로
실제 시장 가격과 다른 값을 반환하도록 조작되는 것.
```

**유형:**

**A. AMM 기반 Oracle 조작 (저유동성 AMM 공격):**
```
낮은 유동성 AMM을 Oracle로 사용하는 경우:
공격자 → 대규모 스왑으로 AMM 내 가격 왜곡
       → 왜곡된 가격을 Oracle로 읽는 프로토콜
       → 조작된 가격 기준으로 과도한 대출/청산 발생
```

**B. 가격 피드 지연 조작:**
```
오프체인 Oracle의 업데이트 지연 시:
시장 가격은 급락했지만 Oracle은 아직 높은 가격 반환
→ 만료된 담보 가치 기준으로 대출 실행
→ 프로토콜 손실
```

**C. 멀티 Oracle 중 하나 조작:**
```
여러 Oracle의 중간값을 사용하지만 임계값 설정이 낮으면:
하나의 Oracle 조작으로 중간값이 왜곡될 수 있음
```

**발생 가능 구간:**
- Oracle 가격 조회 구간
- 담보 평가 구간
- 청산 트리거 판단 구간

## 6) 리스크 유형 × 서비스 유형 매핑

| 리스크 유형 | AMM | Lending | Bridge | Custody |
|---|---|---|---|---|
| 비영구적 손실 | ★★★ | - | - | - |
| 유동성 고갈 | ★★★ | ★★ | - | - |
| Oracle 조작 | ★ | ★★★ | - | - |
| 청산 부족 | - | ★★★ | - | - |
| 연쇄 청산 | - | ★★★ | - | - |
| 검증자 담합 | - | - | ★★★ | - |
| Lock 컨트랙트 취약점 | - | - | ★★★ | - |
| Hot Wallet 해킹 | - | - | - | ★★★ |

## 용어 보충 (Quick Reference)
- **재진입 공격 (Reentrancy)**: 컨트랙트 상태 업데이트 전 외부 호출 시, 수신 컨트랙트가 재귀적으로 다시 호출하여 추가 자산을 탈취하는 공격.
- **플래시론 (Flash Loan)**: 한 트랜잭션 내에서 담보 없이 자산을 빌리고 갚는 메커니즘. 공격에 악용될 수 있음.
- **TWAP (Time-Weighted Average Price)**: 일정 시간 동안의 가격 평균. 순간적인 가격 조작에 저항성이 높음.
- **비영구적 손실 (Impermanent Loss)**: AMM LP가 단순 보유 대비 겪는 손실. 두 자산 가격 비율이 크게 변할수록 증가.
- **연쇄 청산 (Liquidation Cascade)**: 하나의 청산이 추가적인 가격 하락을 유발하고, 더 많은 청산을 일으키는 연쇄 반응.
- **MEV (Maximal Extractable Value)**: 블록 생성자가 트랜잭션 순서를 조작하여 추출할 수 있는 가치. 프론트러닝, 샌드위치 공격 포함.

## 헷갈리는 개념 보충
- 플래시론 자체는 공격이 아니다. 아비트라지, 담보 교환 등 합법적 용도가 있다. "플래시론 공격"은 플래시론을 도구로 사용한 가격 조작 또는 거버넌스 공격이다.
- 재진입 공격은 ETH 전송에서만 발생하지 않는다. ERC-20 토큰 전송 후 콜백이 가능한 토큰(ERC-777 등)에서도 발생할 수 있다.
- 가격 조작과 Oracle 조작은 같은 개념이 아니다. 가격 조작은 시장에서 실제 가격을 움직이는 것이고, Oracle 조작은 Oracle이 읽는 값을 왜곡하는 것이다. 두 방법이 결합될 수 있다.
