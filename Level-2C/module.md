# Level 2C – Advanced Structure Operator  
## 커리큘럼 전체 구성 (Internal Curriculum Spec v1.0)

---

## 레벨 개요

- 레벨 목표:  
  여러 WEB3 구성 요소를 **조합하여 실행하고**,  
  그 실행의 **연쇄 효과를 인지할 수 있는 사람** 만들기
- 레벨 성격:  
  필수 ❌ / 선택 ⭕ / Level 3 진입 검증용
- 핵심 질문:  
  “이 실행이 구조 전체에 어떤 영향을 주는가?”

---

## Level 2C의 위치 정의

Level 2C는 **새로운 기술을 배우는 단계가 아니다.**

- 개발자 ❌
- 설계자 ❌
- 단순 실행자 ❌

👉 **“복합 실행의 결과를 예측할 수 있는 실행자”**

📌 Level 3으로 올라갈 수 있는지 판단하기 위한 **관문 레벨**

---

## 전체 구조

- 총 강의 수: 12강
- 구성: 4개 모듈 × 각 3강
- 설계 원칙:
  - 새로운 개념 최소화
  - “연결”과 “순서”에 집중
  - 실행 이후의 파급 효과를 반드시 추적

---

# Module A. 다중 요소 실행 이해

## 1강. 하나의 실행은 하나로 끝나지 않는다

- 목표:
  - 실행의 연쇄 효과 인식
- 다루는 내용:
  - 하나의 트랜잭션 → 여러 상태 변화
- 통과 기준:
  - 실행 결과가 단일하지 않음을 설명 가능

---

## 2강. 여러 컨트랙트가 엮이는 구조

- 목표:
  - 컨트랙트 간 관계 이해
- 다루는 내용:
  - 호출 → 내부 호출
  - 의존성 구조
- 통과 기준:
  - 컨트랙트 간 관계 1개 이상 설명 가능

---

## 3강. 실행 순서의 중요성

- 목표:
  - 순서가 결과를 바꾸는 구조 이해
- 다루는 내용:
  - 먼저 해야 할 것 / 나중에 해야 할 것
- 통과 기준:
  - 실행 순서 오류 사례 설명 가능

---

# Module B. 멀티 네트워크 실행

## 4강. 네트워크는 독립된 세계다

- 목표:
  - 멀티 네트워크 감각 확보
- 다루는 내용:
  - 체인별 상태 분리
- 통과 기준:
  - 체인 간 상태가 공유되지 않음을 설명 가능

---

## 5강. 네트워크 이동 시 고려 요소

- 목표:
  - 단순 전환이 아닌 “이동” 인식
- 다루는 내용:
  - 브릿지 개념
  - 이동 시간 / 위험
- 통과 기준:
  - 네트워크 이동 리스크 1개 이상 설명 가능

---

## 6강. 멀티 네트워크 실행 실습

- 목표:
  - 네트워크 간 실행 경험
- 실습:
  - A 네트워크 실행 → B 네트워크 후속 행동
- 통과 기준:
  - 멀티 네트워크 실행 1회 이상 성공

---

# Module C. 반복·조건·자동화 감각

## 7강. 반복 실행의 위험과 필요성

- 목표:
  - 반복 실행의 양면성 이해
- 다루는 내용:
  - 실수 누적
  - 가스 비용 누적
- 통과 기준:
  - 반복 실행 리스크 설명 가능

---

## 8강. 조건 기반 실행 이해

- 목표:
  - “아무 때나 실행”에서 탈피
- 다루는 내용:
  - 특정 조건이 충족될 때만 실행
- 통과 기준:
  - 조건 없는 실행의 위험 설명 가능

---

## 9강. 자동화는 ‘위임’이다

- 목표:
  - 자동화의 본질 인식
- 다루는 내용:
  - 자동 실행 = 권한 위임
- 통과 기준:
  - 자동화와 수동 실행의 책임 차이 설명 가능

---

# Module D. Level 3 진입 검증

## 10강. 실행 결과 해석

- 목표:
  - 결과를 숫자가 아닌 “구조 변화”로 보기
- 다루는 내용:
  - 누가 이득 / 손해
  - 구조적 영향
- 통과 기준:
  - 실행 결과를 구조 관점에서 설명 가능

---

## 11강. 이 실행은 필요한가?

- 목표:
  - 실행 자체를 의심하기
- 핵심 질문:
  - 이 실행은 꼭 필요했는가?
- 통과 기준:
  - 불필요한 실행 사례 1개 이상 제시 가능

---

## 12강. Level 3로 넘어갈 자격 질문

- 목표:
  - 사고 전환
- 핵심 질문:
  - 이 문제에 WEB3가 필요한가?
- 통과 기준:
  - 구조 선택 질문 1개 이상 제시 가능

---

## Level 2C 인증 요건

### 인증 방식
**복합 실행 과제 제출**

### 필수 조건
- 2개 이상 컨트랙트 연계 실행
- 2개 이상 네트워크 관여
- 실행 순서 명시
- 실행 결과 구조적 해석 포함

📌 실패해도 됨  
📌 해석과 판단이 핵심

---

## Level 2C 종료 상태 정의

> 나는 여러 실행을 연결할 수 있고,  
> 그 실행이 구조 전체에  
> 어떤 영향을 주는지 설명할 수 있다.

---

## Level 3 안내

- Level 3부터는 더 이상 “잘 실행하는가”를 보지 않는다.
- “애초에 실행해야 했는가?”를 묻는다.
