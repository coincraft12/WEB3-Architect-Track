# Level 2B – Structure Operator  
## 커리큘럼 전체 구성 (Internal Curriculum Spec v1.0)

---

## 레벨 개요

- 레벨 목표:  
  WEB3 구조 안에서 **정해진 역할을 실제로 수행할 수 있는 실행자** 양성
- 레벨 성격:  
  사용자 ❌ / 개발자 ❌ / 구조 실행자 ⭕
- 핵심 전환점:  
  “보내기” → “호출하기”  
  “당황” → “원인 파악”

---

## Level 2B의 위치 정의

Level 2B는 **기술을 만드는 단계가 아니라,  
이미 존재하는 구조를 ‘정확히 사용하는 단계’**다.

- 개발자처럼 창조하지 않는다
- 사용자처럼 무작정 누르지 않는다
- **주어진 역할을 정확히 수행한다**

---

## 전체 구조

- 총 강의 수: 20강
- 구성: 5개 모듈 × 각 4강
- 설계 원칙:
  - 코드는 ‘읽기’까지만
  - 모든 행동에는 “왜 실패했는지”가 따라붙는다
  - 성공보다 **실패 처리 능력**이 중요하다

---

# Module A. 트랜잭션을 이해하는 실행자

## 1강. 트랜잭션은 행동 기록이다

- 목표:
  - 트랜잭션을 결과가 아닌 “기록”으로 인식
- 다루는 내용:
  - Pending / Success / Fail
  - 누가, 언제, 무엇을 요청했는가
- 통과 기준:
  - 트랜잭션 상태 차이 설명 가능

---

## 2강. 가스는 비용이 아니라 조건이다

- 목표:
  - 가스를 “수수료” 개념에서 분리
- 다루는 내용:
  - Gas Limit / Gas Price 개념
  - 왜 실패해도 가스는 나가는가
- 통과 기준:
  - 가스 부족 실패 사례 설명 가능

---

## 3강. 실패 트랜잭션 읽기

- 목표:
  - 실패를 무서워하지 않기
- 실습:
  - 실패 트랜잭션 직접 확인
  - 에러 메시지 위치 찾기
- 통과 기준:
  - 실패 원인 1개 이상 설명 가능

---

## 4강. 성공 트랜잭션 구조 해부

- 목표:
  - 성공 트랜잭션의 공통 구조 인식
- 다루는 내용:
  - From / To / Input Data / Logs
- 통과 기준:
  - 트랜잭션 기본 필드 설명 가능
