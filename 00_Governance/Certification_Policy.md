# Certification Policy (Regulation-Aligned)

본 문서는 `00_Governance/Qualification_Management_and_Operation_Regulations.md`를 단일 기준으로 하며,
모든 교육/평가/운영 문서는 본 정책을 벗어날 수 없다.

## 1. 등급 체계
- Basic
- Associate
- Professional

## 2. 등급별 직무 기준
- Basic: 중앙화 구조와 분산 신뢰 구조의 차이를 설명하고, 지갑/주소/트랜잭션 등 기본 요소를 이해하여 WEB3 구조를 읽는다.
- Associate: 서비스/프로젝트 구조를 WEB3 관점에서 해석하고, 사용자 온보딩/자산흐름/권한-책임 구조를 점검하여 위험요소를 식별한다.
- Professional: 보안/거버넌스/규제/성장 목표에 맞춰 WEB3 구조를 설계/개선하고 핵심 리스크를 통제하는 설계안을 제시한다.

## 3. 검정 기준 원칙
- Basic: WEB3 핵심 구성요소 개념 이해 + 중앙화/탈중앙화 구조 차이 설명 능력
- Associate: 시나리오에서 역할/권한/자산흐름 파악 + 구조 취약점 식별 및 개선 방향 제시
- Professional: 복합 시나리오에서 설계 목표/제약 정리 + 보안/거버넌스/규제/운영 관점의 설계안 구성 및 평가

## 4. 응시자격
- Basic: 제한 없음
- Associate: Basic 취득자
- Professional: Associate 취득자

## 5. 정합성 통제 규칙
- 등급명, 직무내용, 검정방법, 응시자격은 규정 문구와 동일 의미를 유지한다.
- 신규 문서 작성 시 위 항목과 상충하는 표현이 있으면 해당 문서를 수정 또는 폐기한다.
- 레거시 문서는 운영 기준으로 사용하지 않는다.
