# ch3. 권한 집중 구조 점검 - Quiz

## 객관식 퀴즈 (4지선다)

### 문제 1
권한-책임 불일치의 정의로 옳은 것은?
A. 권한도 책임도 모두 사용자에게 있는 상태
B. 권한은 큰데 결과 비용 부담은 약한 상태
C. 권한이 전혀 없는 상태
D. 기술 스택이 다른 상태
- 정답: B
- 해설: ch3 핵심은 권한과 책임의 비대칭에서 구조 리스크가 발생한다는 점이다.

### 문제 2
동결(Freeze)에 대한 설명으로 맞는 것은?
A. 자산 소각과 동일하다
B. 상태 전이를 제한해 자산/행동을 멈추는 통제다
C. 합의 알고리즘의 종류다
D. 지갑 설치 과정이다
- 정답: B
- 해설: 동결은 소유 소멸이 아니라 이동/사용 제한이다.

### 문제 3
차단(Block/Restrict)의 사례로 가장 적절한 것은?
A. 로그인 자체를 막거나 특정 기능만 제한
B. 블록 생성 시간 단축
C. 토큰 발행량 증가
D. 키 길이 확장
- 정답: A
- 해설: 차단은 접근/행동 범위를 제어하는 운영 권한 패턴이다.

### 문제 4
규칙변경(Change Rules)의 리스크로 적절한 것은?
A. 예측 가능성 붕괴
B. 주소 길이 증가
C. 합의 노드 감소
D. 개인키 자동 백업
- 정답: A
- 해설: 사용자의 기존 의사결정 전제가 사후에 흔들릴 수 있다.

### 문제 5
권한 지도(Power Map)의 목적은?
A. UI 컴포넌트 배치
B. 권한-책임 불일치를 구조적으로 가시화
C. 토큰 가격 예측
D. 블록 압축
- 정답: B
- 해설: 누가 승인/거부/변경/책임을 지는지 맵핑해 리스크를 찾는다.
