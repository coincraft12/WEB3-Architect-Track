# ch7. 지갑 개념 정의 - Quiz

## 객관식 퀴즈 (4지선다)

### 문제 1
지갑의 구조적 정의로 가장 적절한 것은?
A. 코인을 보관하는 통
B. 키 관리 + 정책 + 행동 인터페이스
C. 합의 노드
D. 브릿지 서버
- 정답: B
- 해설: 지갑은 단순 보관함이 아니라 서명/정책 도구다.

### 문제 2
수탁형 지갑의 특징은?
A. 사용자가 키를 직접 보관
B. 서비스가 키를 보관/통제
C. 키가 필요 없음
D. 가스가 없음
- 정답: B
- 해설: 수탁형은 편의성이 있지만 통제권이 서비스에 집중된다.

### 문제 3
비수탁형 지갑의 핵심 책임은?
A. 운영사가 복구 보장
B. 사용자가 키/백업 책임을 진다
C. 체인이 자동 복구
D. 주소가 자동 숨김
- 정답: B
- 해설: 통제권이 사용자에게 오는 만큼 책임도 함께 이동한다.

### 문제 4
지갑과 주소의 관계로 맞는 것은?
A. 항상 1:1 고정
B. 지갑은 여러 주소를 다룰 수 있다
C. 주소가 지갑을 생성한다
D. 주소는 오프체인 개념
- 정답: B
- 해설: 지갑은 키 기반으로 여러 주소/계정을 관리할 수 있다.

### 문제 5
승인/권한 관리가 지갑 기능에 포함되는 이유는?
A. UI 장식용
B. 사용자 행동 리스크를 제어하기 위해
C. 수수료를 높이기 위해
D. 합의를 대체하기 위해
- 정답: B
- 해설: 지갑은 서명/승인 경고를 통해 실수를 줄이는 정책 인터페이스다.
