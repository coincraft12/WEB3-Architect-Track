# ch6. 트랜잭션 구조 정의 - Quiz

## 객관식 퀴즈 (4지선다)

### 문제 1
트랜잭션의 본질로 가장 정확한 것은?
A. 단순한 송금 기록
B. 서명된 상태 변경 요청
C. 지갑 설치 로그
D. 정책 공지 문서
- 정답: B
- 해설: 트랜잭션은 실행 전에는 명령이며 실행 후 기록이 남는다.

### 문제 2
Nonce의 역할은?
A. 가스 가격 결정
B. 동일 계정 트랜잭션 순서 보장
C. 주소 생성
D. 이벤트 압축
- 정답: B
- 해설: nonce는 재전송/순서 문제를 통제하는 핵심 필드다.

### 문제 3
EOA->Contract 호출에서 주로 담기는 필드는?
A. Data(함수/파라미터)
B. 주민번호
C. 블록 높이
D. 감사 의견
- 정답: A
- 해설: 컨트랙트 호출은 data 필드가 의미를 결정한다.

### 문제 4
Receipt에 포함되는 정보로 적절한 것은?
A. 사용자의 메모
B. status/gasUsed/logs
C. 키 백업 경로
D. 웹사이트 방문 기록
- 정답: B
- 해설: receipt는 실행 결과 요약 레코드다.

### 문제 5
"실패 트랜잭션도 데이터다"가 맞는 이유는?
A. 실패는 자동 삭제된다
B. 실패 원인과 실행 조건 단서가 남기 때문
C. 실패는 수수료가 없기 때문
D. 실패는 주소를 바꾸기 때문
- 정답: B
- 해설: 실패는 원인 분석/리스크 관리에 중요한 입력값이다.
