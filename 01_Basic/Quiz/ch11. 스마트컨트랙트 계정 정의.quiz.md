# ch11. 스마트컨트랙트 계정 정의 - Quiz

## 객관식 퀴즈 (4지선다)

### 문제 1
스마트컨트랙트의 정의로 가장 적절한 것은?
A. 자동 의사결정 인간
B. 호출 시 실행되는 코드 계정
C. 중앙 서버 백업
D. 수수료 계산기
- 정답: B
- 해설: 컨트랙트는 주소+코드+상태를 가진 계정이다.

### 문제 2
EOA와 Contract 계정의 차이로 맞는 것은?
A. EOA는 코드, Contract는 키
B. EOA는 키 기반, Contract는 코드 기반 통제
C. 둘 다 동일
D. 둘 다 키가 필요 없음
- 정답: B
- 해설: EOA는 개인키 서명, 컨트랙트는 코드 규칙 실행이 핵심이다.

### 문제 3
"컨트랙트는 자동 실행"이 오해인 이유는?
A. 컨트랙트는 주소가 없어서
B. 호출 트리거가 있어야 실행되기 때문
C. 가스가 없어서
D. 토큰이 없어서
- 정답: B
- 해설: 외부/내부 호출이 있어야 상태 전이가 발생한다.

### 문제 4
컨트랙트의 스토리지(Storage)는 무엇을 담는가?
A. 사용자의 브라우저 캐시
B. 컨트랙트의 영구 상태 데이터
C. 체인 이름
D. 시험 점수
- 정답: B
- 해설: 상태 변수는 스토리지에 저장되어 이후 호출에 영향을 준다.

### 문제 5
이벤트(Event/Log)의 용도로 맞는 것은?
A. 상태를 직접 바꾼다
B. 실행 결과 신호를 외부 분석 가능하게 남긴다
C. 키를 생성한다
D. 합의를 대체한다
- 정답: B
- 해설: 이벤트는 상태 자체보다 관측/추적을 돕는 출력 신호다.
