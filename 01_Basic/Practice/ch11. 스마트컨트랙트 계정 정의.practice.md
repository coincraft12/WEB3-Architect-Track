# ch11. 스마트컨트랙트 계정 정의 - Practice

## 미니 실습(20~25분) — 컨트랙트 주소를 “규칙 엔진”으로 해독하기
### 실습 주제: 컨트랙트 1개를 골라 규칙 지도 만들기
(익스플로러에서 가능한 범위에서)

1) 이 주소는 Contract인가? (Contract 표시 확인)  
2) 주요 함수/행동 3가지는 무엇인가? (예: swap, deposit, withdraw 등)  
3) 이벤트 2개는 무엇을 의미하나?  
4) 사용자 관점 리스크 3개(권한/승인/가격/조건)  
5) 이 컨트랙트가 만드는 “상태 전이”를 한 문장으로 써라

**출력(산출물)**: “컨트랙트 규칙 카드 1장”
- Purpose / Authority / Inputs / State / Outputs / Risks
