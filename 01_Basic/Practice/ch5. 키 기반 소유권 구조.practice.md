# ch5. 키 기반 소유권 구조 - Practice

## 미니 실습(15~20분) — ‘서명’의 의미를 눈으로 확인하기
(특정 지갑/체인 고정 없이, 원리 중심으로 설계)

### 실습 A: 메시지 서명(가능하면)
- “나는 2026년 2월 6일 WEB3 구조 설계자 Basic 5강을 학습했다” 같은 짧은 문장을 서명해본다.
- 관찰 포인트:
  - 서명은 “비밀번호 입력”이 아니라 “키로 만든 증거”라는 느낌
  - 같은 메시지도 서명은 특정 주소에 종속된다는 점(주소 바꾸면 서명 달라짐)

### 실습 B: 트랜잭션 서명 화면 관찰(가상으로라도)
- 수신자/금액/네트워크/수수료가 무엇인지 확인
- “서명은 의사결정”이라는 관점에서:
  - 무엇을 승인하는지(자산 이동? 컨트랙트 호출?)를 말로 설명하기

**출력(산출물)**: “서명은 무엇을 증명하는가?” 5문장 정리
